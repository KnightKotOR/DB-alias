**Система управления метаданными MySQL-баз данных** с поддержкой пользовательских псевдонимов и унифицированного доступа к структуре данных.

Проект реализует централизованное хранилище метаданных для набора внешних MySQL-баз данных. Система автоматически извлекает информацию о схемах, таблицах, колонках и ключах, сохраняет ее в специализированную Metadata DB и предоставляет удобный веб-интерфейс для работы с метаданными.

## О проекте

DB Alias предназначен для автоматизированного сбора, хранения и управления метаданными реляционных баз данных MySQL.

Система подключается к произвольному набору внешних баз данных, анализирует их структуру через `INFORMATION_SCHEMA`, сохраняет результаты в собственное хранилище метаданных и позволяет пользователям назначать удобные псевдонимы базам данных и таблицам.

Архитектура проекта состоит из четырех основных компонентов:

* Внешние MySQL-базы данных.
* Центральное хранилище метаданных (Metadata DB).
* Backend-приложение на FastAPI.
* Клиентское веб-приложение на React.

## Ключевые возможности

* Подключение к нескольким MySQL-базам данных одновременно.
* Автоматическое извлечение метаданных через `INFORMATION_SCHEMA`.
* Хранение информации о:

  * базах данных;
  * таблицах;
  * колонках;
  * первичных ключах;
  * внешних ключах;
  * составных ключах.
* Централизованное хранилище метаданных на MySQL.
* Поддержка пользовательских псевдонимов для баз данных и таблиц.
* Выполнение запросов с использованием назначенных псевдонимов.
* REST API для взаимодействия между клиентом и сервером.
* Веб-интерфейс для управления подключениями и просмотра метаданных.

## Хранилище метаданных

Внутренняя база данных содержит информацию о:

* зарегистрированных базах данных;
* таблицах;
* колонках;
* первичных ключах;
* внешних ключах;
* составных ключах;
* пользовательских псевдонимах.

Для хранения составных ключей используются связи с колонками и их порядковыми позициями внутри ключа.

Внешние ключи представлены через связи между ключами соответствующих таблиц.

## Извлечение метаданных

Система использует служебную базу данных MySQL `INFORMATION_SCHEMA` для получения информации о структуре подключаемых БД.

Извлекаются:

* список таблиц;
* список колонок;
* типы данных;
* первичные ключи;
* внешние ключи;
* связи между таблицами.

## Установка и запуск

### Предварительные требования

* Python 3.11+
* MySQL Server
* Node.js
* npm

### Клонирование репозитория

```bash
git clone https://github.com/KnightKotOR/DB-alias.git
cd DB-alias
```

### Backend

```bash
pip install -r requirements.txt
```

Запуск сервера:

```bash
uvicorn app.main:app --reload
```

### Frontend

```bash
cd frontend

npm install
npm start
```

## Пример сценария работы

1. Пользователь подключает внешнюю MySQL-базу данных.
2. Backend извлекает метаданные через `INFORMATION_SCHEMA`.
3. Информация сохраняется в Metadata DB.
4. Пользователь назначает псевдонимы таблицам и базам данных.
5. Через веб-интерфейс выполняются запросы с использованием псевдонимов.
6. Результаты отображаются в клиентском приложении.

## Цель проекта

Разработка масштабируемой системы управления метаданными, способной работать с произвольным количеством MySQL-баз данных и предоставлять единый механизм доступа к информации об их структуре.
